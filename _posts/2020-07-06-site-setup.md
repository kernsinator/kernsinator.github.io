Building this site has been a goal of mine for several months. While working on other projects, I had come across static site generators, even building a few sites using Hugo. At the start of this year I also began to familiarize myself with Git and Github. In the end, I settled on a combination of these tools.

Learning about Github led me to Github pages, which allows you to host a website from a Github repository. Upon further googling, I found that [Jekyll](https://jekyllrb.com/), a static site generator written in Ruby, actually powers Github pages. I used [this](http://jmcglone.com/guides/github-pages/) tutorial to get me up and running with Jekyll and Github Pages. Once I had a basic site online and functional, I began to explore Jekyll themes. Lucky for us, [https://jekyllthemes.io/](https://jekyllthemes.io/) exists. I settled on a theme called Minimal Mistakes.

[Minimal Mistakes](https://mmistakes.github.io/minimal-mistakes/) is responsible for basically everything you can see and/or interact with on this site. The page linked above provides everything needed to get an attractive and functional site working in a matter of minutes. Thorough documentation made it easy to include useful features such as author information or the Table of Contents located on the Resources page.

I had one goal from the beginning that was not covered in the above tutorial. That goal was to host my site under a custom domain, tylerkerns.com. If you are reading this, you can look at the address bar in your browser and see that it worked.

Github Pages allows you to host from a repo with the url "username.github.io". It is also possible, with a little effort, to use a custom domain. Github has a section in the [Docs](https://docs.github.com/en/github/working-with-github-pages/configuring-a-custom-domain-for-your-github-pages-site) that walks you through that proess. I encountered no difficulties whatsoever. 

Choosing a domain name was easy, as I used my own name. Choosing a domain registrar, on the other hand, took some research. With my lack of experience on the topic, I found Google Domains to be a good choice. Google Domains has straightforward pricing and plenty of documentation to help you through any sticking points.
