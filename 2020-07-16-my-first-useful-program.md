My first introduction to programming was using Python. That seems to be a particularly common language for people to encounter first. My initial introduction was through Codecademy courses, which I completed without ever applying that knowledge. When I reapproached the topic, I worked through Crash Course Python, as it also included real projects. This left me with a concrete product after all that reading. I then did not do any programming for a good while. After some time had passed, I decided to learn to code for real. To motivate myself, I would need an actual useful project, but one simple enough that I could figure it out with my very rudimentary skill. I thought about things in my life that could stand to be automated and one option stood out to me in particular. My work schedule.

Prior to integrating my work calendar with the calendar app on my phone, I always utilized a web browser to see my upcoming shifts. Working such an irregular schedule leads to frequent checks. Just accessing the main page of my work schedule take nearly 60 seconds of clicking and waiting.

I initially hardcoded data, before using input vairable. Though still not secure.
